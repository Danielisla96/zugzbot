# Plan: Enhanced SDD Sidebar Plugin

## Goal
Enhance the existing `sdd-sidebar.tsx` with rich real-time SDD cycle information, following opencode-quota's plugin architecture patterns.

## Approach
- **Local files** (not npm-published), symlinked via `install-plugin.sh`
- **TUI-only** (no server plugin / slash commands)
- Keep both `.tsx` (JSX) and `.ts` (hyperscript) versions
- Modular code under new `plugin/sdd-plugin/` directory

## Architecture (inspired by opencode-quota)

### File Structure
```
plugin/
├── package.json                      # MODIFIED: add @opentui/* deps
├── sdd-sidebar.tsx                   # MODIFIED: re-export from sdd-plugin/tui.tsx
├── sdd-sidebar.ts                    # MODIFIED: re-export from sdd-plugin (hyperscript)
└── sdd-plugin/                       # NEW
    ├── tui.tsx                       # Slot registration + default export
    ├── tui.ts                        # Same, hyperscript version
    ├── lib/
    │   ├── types.ts                  # SddLock, PhaseHistoryEntry, TaskItem, etc.
    │   ├── constants.ts              # SDD_PHASES, AGENT_MAP, POLL_INTERVAL
    │   ├── reader.ts                 # All file I/O (lock, history, tasks)
    │   ├── state.ts                  # Signals, refresh orchestration, SddResource
    │   ├── format.ts                 # Progress bars, display helpers
    │   └── styles.ts                 # Theme-aware color helpers
    └── components/
        ├── CollapsibleSection.tsx     # Reusable expand/collapse with KV persistence
        ├── SidebarPanel.tsx           # Main sidebar composing all sections
        ├── PhaseList.tsx             # Phase progress checklist
        ├── TaskChecklist.tsx         # Tasks from orchestrator_tasks.md (parsed)
        ├── PhaseHistory.tsx          # Timeline from phase_history.jsonl
        └── CompactStatus.tsx         # session_prompt + home_bottom compact views
```

### Component Tree
```
tui.tsx
├── sidebar_content → <SidebarPanel api={api} sessionID={id} />
│   ├── "[SDD Monitor] • v2.0" (header)
│   ├── CollapsibleSection "SDD Overview"
│   │   ├── Change name, Mode (INTERACTIVO/PILOTO AUTOMÁTICO)
│   │   ├── Status (IDLE / RUNNING ⚡), Iteration count
│   │   └── Progress bar: [████████░░░░░░░░] 37%
│   ├── CollapsibleSection "Phase Progress"
│   │   └── PhaseList: ✓/⚡/▶/○ icons for each phase
│   ├── CollapsibleSection "Agent Activity"
│   │   ├── Current agent badge @sdd-implementer
│   │   ├── AgentTimeline (last 5 transitions)
│   │   └── PhaseHistory (full timeline when expanded)
│   └── CollapsibleSection "Task Checklist"
│       └── TaskChecklist: [x]/[ ] items from orchestrator_tasks.md
├── session_prompt → <CompactStatus text="F3 @sdd-implementer | ⚡ RUNNING" />
└── home_bottom → <HomeBottom text="SDD: nuevo-cambio • F3/8 • ⚡ in_progress" />
```

### Data Flow
```
reader.ts → fs.readFile → signals in state.ts → SolidJS components
  - readLock(): .openspec/sdd-lock.json (poll 2s)
  - readHistory(): .openspec/changes/<name>/phase_history.jsonl
  - readTasks(): .openspec/changes/<name>/orchestrator_tasks.md
  - resolveWorkspaceDir(): api.state.path.worktree || cwd()
```

### KV Persistence (collapsed states)
- `sdd-overview-collapsed`, `sdd-phases-collapsed`
- `sdd-agents-collapsed`, `sdd-tasks-collapsed`

## Implementation Steps

### Step 1: Create `lib/types.ts` and `lib/constants.ts`
- Types: `SddLock`, `PhaseHistoryEntry`, `TaskItem`, `SddSidebarState`, `SddCompactState`
- Constants: `SDD_PHASES` array, `SLOT_ORDER`, `POLL_INTERVAL_MS` (2000ms), mount recovery delays

### Step 2: Create `lib/reader.ts`
- `readLockFile(workspaceDir)` → reads + parses sdd-lock.json
- `readPhaseHistory(workspaceDir, changeName)` → reads JSONL, returns last 50
- `readTasks(workspaceDir, changeName)` → parses markdown checklist ([x]/[ ])
- All with try/catch, return null/[] on error

### Step 3: Create `lib/format.ts` and `lib/styles.ts`
- `getProgressBar(phaseNum)`: ██/░░ rendering
- `formatPhaseName(phase)`, `formatSubagent(agent)`, `formatTimestamp(iso)`
- `getPhaseColor(phaseNum, isActive, isCompleted, theme)` helpers

### Step 4: Create `lib/state.ts`
- `createSddResource(api, sessionID)` → manages signals, refresh loop, event listeners
- `useSessionSddResource(api, sessionID)` → hook for components
- WeakMap-based lifecycle management
- Poll every 2s via setInterval
- Listen for `session.updated`, `message.updated` for immediate refresh
- Mount recovery retries at 500ms, 1.5s, 4s

### Step 5: Create `components/CollapsibleSection.tsx`
- Reusable expand/collapse with clickable header (▶/▼)
- KV persistence for collapsed state
- Optional summary text shown when collapsed
- Props: `id, title, children, api, summary?`

### Step 6: Create `components/PhaseList.tsx`
- Renders all 9 phases with ✓/⚡/▶/○ status icons
- Active phase shown bold with @subagent annotation

### Step 7: Create `components/TaskChecklist.tsx`
- Parses orchestrator_tasks.md for section headers and [x]/[ ] items
- Shows completion ratio (e.g. 3/12 completadas)
- Items listed with [x] or [ ] with appropriate dimming

### Step 8: Create `components/PhaseHistory.tsx`
- Reads phase_history.jsonl entries
- Shows as simple timeline: `F2 → F3 @sdd-implementer (2026-05-23)`
- Limited to last 10 entries for sidebar compactness

### Step 9: Create `components/CompactStatus.tsx`
- `CompactStatusLine`: one-line text for session_prompt
- `HomeBottomView`: one-line text for home_bottom

### Step 10: Create `components/SidebarPanel.tsx`
- Composes all CollapsibleSections
- Handles idle/no-change state gracefully
- Bottom divider

### Step 11: Create `tui.tsx` and `tui.ts`
- Register slots: `sidebar_content` (order: 10), `session_prompt`, `home_bottom`
- Default export as `TuiPluginModule`

### Step 12: Update `sdd-sidebar.tsx`
- Re-export from `./sdd-plugin/tui`

### Step 13: Create hyperscript equivalents
- For each new .tsx file, create .ts version using `solid-js/h`

### Step 14: Update `package.json`
- Add `@opentui/core` and `@opentui/solid` to `dependencies` (^0.2.7)
- Add `solid-js` to `dependencies` (^1.8.0)
- Update exports if needed
